# Binary search trees

A binary search tree keeps values in order by shape: every node is larger than everything in its left subtree and smaller than everything in its right subtree. That rule lets a search go left or right at each node instead of checking everything. The figure below builds trees and traces the insert and search walks so you can watch the path, and see how the insert order decides whether the tree stays short or collapses into a line.
