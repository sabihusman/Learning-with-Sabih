# Dynamic programming

Dynamic programming speeds up a recursive computation by remembering the answers to subproblems instead of solving them again. Naive recursive Fibonacci is the classic example: it solves the same smaller problems over and over, so its call tree explodes. Memoizing each result once collapses that tree dramatically.
