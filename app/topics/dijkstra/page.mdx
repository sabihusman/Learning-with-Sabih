# Dijkstra's shortest path

When a graph's edges carry weights, like distances or costs, the shortest path is the one with the smallest total weight, which is not always the one with the fewest hops. Dijkstra's algorithm finds the cheapest path from a source to every other node by repeatedly locking in the nearest unfinalized node and relaxing its edges.
